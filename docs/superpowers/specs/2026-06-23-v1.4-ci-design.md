# v1.4 CI Pipeline — Design

**Date**: 2026-06-23  
**Status**: Approved (implementing)  

## Problem

Project has no CI. Every contributor and reviewer must manually run the 4 gates (fmt, clippy, test, build). With no `.github/workflows/`, there's no automated safety net for PRs.

## Design

### Workflow: `ci.yaml`

| Property | Value |
|---|---|
| Trigger | `push: [main]` + `pull_request: [main, milestone/*]` |
| Runner | `ubuntu-latest` |
| Rust | `stable` (components: clippy, rustfmt) |
| Strategy | Single job, 4 steps, fail-fast |

### Steps (in order)

1. `cargo fmt --check`
2. `cargo clippy --all-targets -- -D warnings`
3. `cargo test`
4. `cargo build --release`

### Files Changed

| File | Action |
|---|---|
| `.github/workflows/ci.yaml` | Create |

## Non-Changes

- No cache configuration (project is small, build time ~3min)
- No matrix strategy (complexity not justified)
- No special system dependencies
- No changes to existing PR template or any source code

## Verification

- Workflow syntax validated via `actionlint` or GitHub web UI

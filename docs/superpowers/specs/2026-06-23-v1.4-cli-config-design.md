# v1.4 CLI 交互式配置 + 入口重设计 — Spec

**Date**: 2026-06-23  
**Status**: Approved  

## 问题

当前 TUI 渲染空白、Wizard 不可用。用户需要手动写 YAML 文件才能配置 LLM，没有纯文本交互式配置路径。

## 设计

### 一、命令入口重设计

| 命令 | 行为 |
|---|---|
| `eflow` | 检测 LLM 配置 → 未配则进 CLI 文本配置 → 配好进 **CLI 对话** |
| `eflow tui` | 检测 LLM 配置 → 未配则进 CLI 文本配置 → 配好进 **TUI 对话** |
| `eflow --execute "..."` | 保留（单次执行） |
| `eflow --show-config` | 保留 |
| `eflow --list-profiles` | 保留 |
| `eflow session start` | 保留（headless NDJSON） |
| `eflow init` | 保留（TUI Wizard，开发者用） |

子命令：在 `Cli` 的 `Command` 枚举中新增 `Tui` 变体。

### 二、配置检测

```
启动 → 扫 ~/.eflow/providers/*.yaml
  ├─ 为空 → 进文本配置流程
  └─ 有文件 → 进对话模式
```

检测在 `main.rs` 中完成，两个入口（`eflow` 和 `eflow tui`）共享同一段代码。

### 三、CLI 文本菜单组件

```rust
    pub label: String,         // 显示文本
}

    .....

```

交互：
- `↑` / `k` — 上移
- `↓` / `j` — 下移
- `1-9` — 直接跳到第 N 项
- `Enter` — 确认当前选中
- `Esc` — 取消（返回 None）

### 四、文本配置 6 步流程

| 步 | 内容 | 交互 |
|---|---|---|
| 1. 欢迎 | "未检测到 LLM 配置，进入配置向导" | Enter 继续 |
| 2. 选厂商 | 7 个选项（6 预设+1 自定义） | ↑↓ + 数字 |
| 3. 填 Key | "API Key:"，掩码输入 | 打字 + Enter |
| 4. 拉模型 | "正在拉取可用模型..." | 自动 |
| 5. 选模型 | 列表选择 | ↑↓ + 数字 |
| 6. 确认 | 摘要确认 | Enter/Y / Esc/N |

### 六、文件写入

配置确认后：
1. 写 `~/.eflow/providers/{id}.yaml`（provider 配置）
2. 更新 `eflow.yaml` 的 `llm.routing` 指向新 provider
3. 提示"配置完成，重新运行 eflow"

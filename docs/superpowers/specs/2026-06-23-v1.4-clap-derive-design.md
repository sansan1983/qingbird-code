# v1.4 Clap Derive Subcommand — Design

**Date**: 2026-06-23  
**Status**: Approved (implementing)  

## Problem

`eflow init` and `eflow session start` are routed via hand-written `std::env::args()` parsing in `main.rs`, including a hand-written `parse_session_flag` helper. The `Cli` struct already uses clap derive — only the subcommand routing is manual.

## Design

### Add Subcommand enums to `cli.rs`

```rust
#[derive(Subcommand)]
pub enum Command {
    Init,
    Session { #[command(subcommand)] action: SessionAction },
}

#[derive(Subcommand)]
pub enum SessionAction {
    Start { #[arg(long)] config: Option<String>, #[arg(long)] lang: Option<String> },
}
```

### Add to `Cli` struct

```rust
pub struct Cli {
    #[command(subcommand)]
    pub command: Option<Command>,
    // ...existing flags unchanged
}
```

### Changes to `main.rs`

- Remove L40-71: hand-written `args.get(1)` routing for `init` and `session start`
- Remove L281-299: `parse_session_flag` function
- Replace `Cli::parse_args()` with `Cli::parse()`
- Add `match cli.command { ... }` to dispatch to `cli::init::run()` / `cli::start::run()`

## Files Changed

| File | Change |
|---|---|
| `src/interaction/cli.rs` | Add `Command` + `SessionAction` enums, add `command` field to `Cli` |
| `src/main.rs` | Remove hand-written routing, add subcommand match |

## Non-Changes

- No breaking changes to existing CLI interface
- No changes to `cli::init::run()` or `cli::start::run()` signatures
- No changes to test code (tests use `Cli::parse_from()` which remains compatible)
- No Cargo.toml changes (clap already in deps)

## Verification

- `cargo test` passes (existing CLI tests validate all flags still work)
- `cargo clippy` zero warnings
- Manual: `eflow init`, `eflow session start`, `eflow --execute`, `eflow --show-config` all parse correctly

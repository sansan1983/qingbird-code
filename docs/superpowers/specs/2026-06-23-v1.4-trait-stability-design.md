# v1.4 Trait Stability Check — Design

**Date**: 2026-06-23  
**Status**: Approved (implementing)  

## Problem

`LlmProvider` trait is frozen since v1.3 (ADR-0011). Breaking changes require ADR + CHANGELOG. Without CI enforcement, an accidental signature change could go unnoticed.

## Design

Create `.github/workflows/check-trait-stability.yml` triggered on PRs modifying `src/infrastructure/llm/types.rs`.

### Check Logic

1. Checkout both PR branch and `origin/main`
2. Extract `fn` lines from `LlmProvider` trait definition in both versions
3. Compare the set of method signatures (name + parameters)
4. 4 abstract methods (`chat`, `chat_stream`, `supports_prefix_cache`, `name`) must match exactly
5. 2 default methods (`list_models_endpoint`, `retry_params`) are exempt

### Script

```bash
# Extract abstract method signatures from trait (lines before first default fn)
awk '/pub trait LlmProvider/,/^}/' types.rs | grep 'fn ' | grep -v 'list_models_endpoint\|retry_params' | sed 's/.*fn //' | sed 's/;.*//' | sort
```

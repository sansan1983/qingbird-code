# eflow v1.3.2 实施计划 — CLI 契约（headless 跨进程接口）

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 v2.0 GUI 套壳铺路：在 spec A (LLM 抽象) + spec B1 (SlashCommand trait) 之上，加 headless CLI 契约层——`eflow session start` 持续运行 + stdin JSON 协议 + 6+1 个事件 schema 冻结 + 4 档 exit code + Python 套壳集成测试。TUI 零改造（仍走 spec B1 同进程 trait dispatch）。

**Architecture:** 7 个阶段循序推进。M1 clap subcommand 树（无 subcommand → TUI，有 → headless）。M2 stdout/stderr 协议（CliOutput + flush + exit code）。M3 `eflow session start` 持续运行（tokio::select 跑 2 个 task + SystemReady 事件）。M4 stdin 协议（5 种 StdinCommand 解析 + EOF 退出 + 非法 JSON 不退出）。M5 `eflow init` 委托 Wizard。M6 SystemReady 事件 + JSON 序列化。M7 契约文档 + Python 集成测试。**TUI 零改造**——main.rs 加 clap 路由，**不**改 tui.rs / concierge.rs。**契约冻结 v1.3.0 起**——事件 schema / stdin action / exit code 不再破坏性变更。

**Tech Stack:** Rust 2024 + tokio + clap（v1.2 已有）+ async-trait + serde + serde_json + reqwest（v1.3.0 已有）+ chrono（需新增 dev-dependency）

**Spec reference:** `docs/superpowers/specs/2026-06-17-eflow-v1.3-b2-cli-contract-design.md`（973 行，3 条 ADR：0016 TUI 零改造 / 0017 契约冻结 / 0018 单 subcommand 模式）

//! stdout/stderr 协议
//!
//! 关键设计决策：
//! - `json` 用 println!（stdout）——**不**flush（GUI 读行前不需强制 flush）
//! - `ndjson_event` 立即 flush——**events 实时性关键**
//! - `info` / `error` 用 eprintln!（stderr）
//! - **不**直接调 tracing——tracing 自己配 tracing_subscriber 走 stderr

use serde::Serialize;

pub struct CliOutput;

impl CliOutput {
    /// 输出 JSON 对象到 stdout（**永远只走 stdout**）
    pub fn json<T: Serialize>(value: &T) -> crate::common::error::Result<()> {
        let json = serde_json::to_string(value)?;
        println!("{}", json);
        Ok(())
    }

    /// 输出一行 NDJSON（events 流用）——立即 flush
    pub fn ndjson_event<T: Serialize>(value: &T) -> crate::common::error::Result<()> {
        let json = serde_json::to_string(value)?;
        println!("{}", json);
        // 立即 flush，让 GUI 实时收到
        use std::io::Write;
        std::io::stdout().flush().ok();
        Ok(())
    }

    /// 输出人类可读消息到 stderr
    pub fn info(msg: &str) {
        eprintln!("{}", msg);
    }

    /// 输出错误消息到 stderr
    pub fn error(msg: &str) {
        eprintln!("error: {}", msg);
    }
}

//! exit code 转换 + Ctrl+C 处理
//!
//! 4 档退出码（契约冻结 v1.3.0 起）：
//! - 0 = ok
//! - 1 = 用户错误（参数非法、文件不存在、KEY 无效等）
//! - 2 = 系统错误（网络失败、文件 IO 错误、内部错误）
//! - 130 = Ctrl+C

use std::process::ExitCode;

use crate::common::error::EflowError;

pub fn exit_code(err: &EflowError) -> ExitCode {
    match err {
        // 用户错误
        EflowError::Config(_) | EflowError::LlmAuthFailed(_) => ExitCode::from(1),

        // 系统错误
        EflowError::LlmProvider(_)
        | EflowError::RateLimited(_)
        | EflowError::Internal(_)
        | EflowError::Io(_) => ExitCode::from(2),

        // 其它默认 1
        _ => ExitCode::from(1),
    }
}

/// Ctrl+C 退出码 130（Unix 习惯）
pub fn handle_sigint() -> ExitCode {
    ExitCode::from(130)
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn config_error_maps_to_user_error_1() {
        let err = EflowError::Config("bad config".into());
        assert_eq!(exit_code(&err), ExitCode::from(1));
    }

    #[test]
    fn auth_failed_maps_to_user_error_1() {
        let err = EflowError::LlmAuthFailed("anthropic".into());
        assert_eq!(exit_code(&err), ExitCode::from(1));
    }

    #[test]
    fn provider_error_maps_to_system_error_2() {
        let err = EflowError::LlmProvider("network down".into());

//! `eflow session start` —— 持续运行的 headless 模式
//!
//! 启动流程：
//! 1. 加载 config
//! 2. 切语言（如果指定）
//! 3. 构造 LlmRouter
//! 4. 构造 CommandRegistry（注册 6 个 builtin 命令）
//! 5. 构造 Concierge
//! 6. 订阅事件通道
//! 7. 输出 SystemReady 事件（**第一行**）
//! 8. tokio::select 跑 2 个 task：
//!    - 任务 A：监听 event channel → 6 个事件 → CliOutput::ndjson_event()
//!    - 任务 B：读 stdin → 5 个 StdinCommand → handler
//! 9. Ctrl+C / EOF / panic 触发主循环退出
//! 10. 退出时 exit_code

use std::path::PathBuf;
use std::process::ExitCode;
use std::sync::Arc;

use tokio::sync::broadcast;

use crate::application::concierge::Concierge;
use crate::common::error::Result;
use crate::infrastructure::config::load_config;
use crate::infrastructure::event::Event;
use crate::infrastructure::llm::router::LlmRouter;
use crate::interaction::slash::builtin::{
    help::HelpCmd, lang::LangCmd, level::LevelCmd, model::ModelCmd, profile::ProfileCmd, quit::QuitCmd,
};
use crate::interaction::slash::CommandRegistry;

use super::error::exit_code;
use super::output::CliOutput;

/// 启动会话并持续运行
pub async fn run(config: Option<PathBuf>, lang: Option<String>) -> ExitCode {
    // 1. 加载 config
    let config_path = config.unwrap_or_else(|| {
        dirs::config_dir()
            .map(|p| p.join("eflow").join("config.yaml"))
            .unwrap_or_else(|| PathBuf::from("./eflow.yaml"))
    });
    let eflow_config = match load_config(&config_path) {
        Ok(c) => c,
        Err(e) => {
            CliOutput::error(&format!("config load failed: {e}"));
            return exit_code(&e);
        }
    };

    // 2. 切语言（如果指定）
    if let Some(ref locale) = lang {
        crate::infrastructure::locale::init(Some(locale));
    }

    // 3. 构造 LlmRouter（spec A）
    let provider_dir = dirs::config_dir()
        .map(|p| p.join("eflow").join("providers"))
        .unwrap_or_else(|| PathBuf::from("./providers"));
    let router = match LlmRouter::from_config(&eflow_config, &provider_dir) {
        Ok(r) => r,
        Err(e) => {
            CliOutput::error(&format!("router init failed: {e}"));
            return exit_code(&e);
        }
    };

    // 4. 构造 CommandRegistry（spec B1 6 个 builtin）
    let mut command_registry = CommandRegistry::new();
    command_registry.register(Arc::new(ModelCmd));
    command_registry.register(Arc::new(ProfileCmd));
    command_registry.register(Arc::new(LangCmd));
    command_registry.register(Arc::new(LevelCmd));
    command_registry.register(Arc::new(HelpCmd::new(&command_registry)));  // v1.3.1 已有
    command_registry.register(Arc::new(QuitCmd));
    if let Err(e) = command_registry
        .required_register(&["model", "profile", "lang", "level", "help", "quit"])
    {
        CliOutput::error(&format!("command registry init failed: {e}"));
        return exit_code(&e);
    }

    // 5. 构造 Concierge
    let mut concierge = Concierge::new(router, command_registry);

    // 6. 订阅事件通道
    let mut event_rx = concierge.subscribe_events();

    // 7. 输出 SystemReady 事件（**第一行**）
    let task_id = concierge.task_id();
    if let Err(e) = CliOutput::ndjson_event(&serde_json::json!({
        "event_type": "SystemReady",
        "task_id": task_id,
        "started_at": chrono::Utc::now().to_rfc3339(),
    })) {
        return exit_code(&e);
    }

    // 8. spawn 2 个 task + tokio::select 等待
    let exit_code = run_two_tasks(&mut concierge, &mut event_rx).await;

    // 9. 优雅退出
    let _ = concierge.shutdown().await;
    exit_code
}

async fn run_two_tasks(
    concierge: &mut Concierge,
    event_rx: &mut broadcast::Receiver<Event>,
) -> ExitCode {
    tokio::select! {
        code = task_a_listen_events(event_rx) => code,
        code = super::stdin::read_loop(concierge) => code,

            Ok(Event::TaskFailed { task_id, error }) => {
                let _ = CliOutput::ndjson_event(&serde_json::json!({
                    "event_type": "TaskFailed",
                    "task_id": task_id,
                    "error": error,
                }));

            Ok(Event::RiskEscalated { task_id, from, to }) => {

                    "event_type": "RiskEscalated",
                    "task_id": task_id,
                    "from": format!("{:?}", from),

//! stdin 协议
//!
//! 5 个 action JSON：send / end / level / lang / help
//!
//! 关键设计决策：
//! - `Send` 的 task_id 可选——GUI 不必先知道 task_id；eflow 自动生成

use crate::application::concierge::Concierge;
use crate::common::error::Result;

#[derive(Deserialize, Debug)]
#[serde(tag = "action", rename_all = "snake_case")]
pub enum StdinCommand {
    Send {
        task_id: Option<Uuid>,
        task: String,
    },
    End {
        task_id: Uuid,
    },
    Level {

/// 持续读 stdin 一行行，dispatch 到 5 个 handler
pub async fn read_loop(concierge: &mut Concierge) -> ExitCode {
    let stdin = tokio::io::stdin();
    let mut reader = BufReader::new(stdin);
    let mut line = String::new();

    loop {
        line.clear();
        match reader.read_line(&mut line).await {
            Ok(0) => return ExitCode::from(0),  // EOF
            Ok(_) => {
                let trimmed = line.trim();
                if trimmed.is_empty() {
                    continue;  // 空行跳过
                }

                match serde_json::from_str::<StdinCommand>(trimmed) {
                    Ok(StdinCommand::Send { task_id, task }) => {
                        let _ = handlers::send::dispatch(concierge, task_id, &task).await;
                    }
                    Ok(StdinCommand::End { task_id }) => {
                        let _ = handlers::end::dispatch(concierge, task_id).await;
                        return ExitCode::from(0);
                    }
                    Ok(StdinCommand::Level { task_id, level }) => {
                        let _ = handlers::level::dispatch(concierge, task_id, &level).await;
                    }
                    Ok(StdinCommand::Lang { task_id, locale }) => {
                        let _ = handlers::lang::dispatch(concierge, task_id, &locale).await;
                    }
                    Ok(StdinCommand::Help) => {
                        let _ = handlers::help::dispatch(concierge).await;
                    }
                    Err(e) => {
                        // 解析失败 → stderr 报错 + **继续**读下一行
                        CliOutput::error(&format!("stdin parse failed: {e}"));
                    }
                }
            }
            Err(e) => {
                CliOutput::error(&format!("stdin read error: {e}"));
                return exit_code(&crate::common::error::EflowError::Io(e));
            }
        }
    }
}

use crate::infrastructure::llm::router::LlmRouter;

pub async fn run() -> ExitCode {
    // 1. 构造 LlmRouter（用 default config + 空 provider dir，Wizard 走 stdin 填）

        .map(|p| p.join("eflow").join("providers"))
        .unwrap_or_else(|| PathBuf::from("./providers"));

    let router = match LlmRouter::from_config(&default_config, &provider_dir) {

        Arc::new(WelcomeStep),
        Arc::new(LanguageStep),
        Arc::new(ProviderStep),
        Arc::new(ProtocolStep),
        Arc::new(ApikeyStep),
        Arc::new(ModelStep),
        Arc::new(ConfirmStep),
    ];
    let wizard = Wizard::new(steps);

    // 3. 调 Wizard.run()（spec B1 已有）
    match wizard.run() {
        Ok(crate::interaction::wizard::WizardOutcome::Completed(_state)) => {
            CliOutput::info("init complete");

# eflow v1.3.2 CLI 契约（v2.0 GUI 套壳接口）

> **契约冻结 v1.3.0 起**（spec B2 ADR-0017）——v1.4+ 变更需走 ADR。

## 概述

GUI 通过 spawn `eflow session start` 进程 + stdio 通信：

```
┌─────────────────┐
│  GUI 进程        │
│  ┌────────────┐  │      spawn       ┌─────────────────┐
│  │  按钮/列表  │──┼─────────────────▶│  eflow 进程      │
│  │  ↓ 转换     │  │                  │  ┌────────────┐  │
│  │  spawn      │  │  stdout: NDJSON  │  │ Concierge  │  │
│  │  eflow      │◀─┼──────────────────│  └────────────┘  │
│  └────────────┘  │  stdin: JSON     └─────────────────┘
└─────────────────┘
```

**两条路径行为 100% 一致**：
- TUI 同进程 `CommandRegistry::dispatch()`（spec B1）
- GUI 跨进程 `eflow session start` + stdin/stdout 协议

**TUI / GUI 共用同一份 `SlashCommand` trait impl**（spec B2 ADR-0016）。

## 1. 进程生命周期

```bash
# 启动
eflow session start --lang en-US
# 输出第一行（stdout）= SystemReady 事件
# 然后持续输出 events NDJSON，stdin 接收指令
# 退出码：0 / 1 / 2 / 130
```

## 2. stdout 协议（NDJSON）

### 2.1 SystemReady 事件（start 后第一行）

```json
{"event_type": "SystemReady", "task_id": "uuid-v4", "started_at": "2026-06-17T12:34:56Z"}
```

### 2.2 6 个用户事件（直接序列化 Event enum）

```json
{"event_type": "TaskStarted", "task_id": "uuid-v4", "description": "review main.rs"}
{"event_type": "TaskCompleted", "task_id": "uuid-v4", "summary": "no issues found"}
{"event_type": "TaskFailed", "task_id": "uuid-v4", "error": "API timeout"}
{"event_type": "RiskEscalated", "task_id": "uuid-v4", "from": "L1", "to": "L2"}
{"event_type": "UserInputRequired", "prompt": "confirm? [y/n]"}
{"event_type": "SystemShutdown"}
```

## 3. stdin 协议

每行 1 条 JSON 指令（**解析失败不退出**）：

```json
{"action": "send", "task": "review main.rs"}
{"action": "send", "task_id": "uuid-v4", "task": "..."}    // task_id 可选 = 自动生成
{"action": "end", "task_id": "uuid-v4"}
{"action": "level", "task_id": "uuid-v4", "level": "simple"}
{"action": "lang", "task_id": "uuid-v4", "locale": "en-US"}
{"action": "help"}
```

## 4. stdout ack（stdin action 收到后输出）

```json
{"ack": "send", "task_id": "uuid-v4", "dispatched_at": "2026-06-17T12:34:57Z"}
{"ack": "end", "task_id": "uuid-v4", "ended_at": "2026-06-17T12:35:00Z"}
{"ack": "level", "task_id": "uuid-v4", "level": "simple"}
{"ack": "lang", "task_id": "uuid-v4", "locale": "en-US"}
{"ack": "help", "commands": []}    // v1.3.2 阶段 commands 字段是空 vec
```

## 5. stderr 协议

人类可读，**GUI 不解析**——仅供用户调试。例：
```
error: config load failed: ...
error: stdin parse failed: ...
error: send failed: ...
```

## 6. exit code

| code | 含义 | 触发 |
|---|---|---|
| 0 | ok | 正常退出（end / EOF / SystemShutdown） |
| 1 | 用户错误 | 参数非法 / 文件不存在 / KEY 无效 / 用户 Esc 取消 init |
| 2 | 系统错误 | 网络失败 / 文件 IO 错误 / 内部错误 |
| 130 | Ctrl+C | 用户按 Ctrl+C |

## 7. Python 套壳示例

```python
import subprocess, json

proc = subprocess.Popen(
    ["eflow", "session", "start", "--lang", "en-US"],
    stdin=subprocess.PIPE,
    stdout=subprocess.PIPE,
    stderr=subprocess.PIPE,
    text=True,
    bufsize=1,  # 行缓冲
)

# 读第一行 SystemReady（GUI 知道 eflow 启动完成）
first_event = json.loads(proc.stdout.readline())
assert first_event["event_type"] == "SystemReady"

# 写 send 指令
proc.stdin.write(json.dumps({"action": "send", "task": "review main.rs"}) + "\n")
proc.stdin.flush()

# 持续读 events
for line in proc.stdout:
    event = json.loads(line)
    if event["event_type"] == "TaskCompleted":
        handle_result(event["summary"])
    elif event["event_type"] == "TaskFailed":
        handle_error(event["error"])

# 写 end 指令
proc.stdin.write(json.dumps({"action": "end", "task_id": task_id}) + "\n")
proc.stdin.flush()
proc.wait()
```

import os

def get_eflow_binary():

    project_root = Path(__file__).parent.parent

def get_test_provider_dir():

    import shutil

    tmp = Path(tempfile.mkdtemp(prefix="eflow-test-"))
    src = Path(__file__).parent / "fixtures" / "providers" / "mock.yaml"
    if src.exists():
        shutil.copy(src, tmp / "mock.yaml")

@pytest.fixture

    config = get_test_config_path()

def read_event(proc, timeout=5):
    """读 stdout 一行，parse JSON event"""
    import select
    start = time.time()
    while time.time() - start < timeout:
        line = proc.stdout.readline()
        if line:
            return json.loads(line)
        time.sleep(0.1)
    raise TimeoutError("read event timeout")


def send_stdin(proc, cmd: dict):
    """写 stdin 一条 JSON 指令"""
    proc.stdin.write(json.dumps(cmd) + "\n")
    proc.stdin.flush()


def test_01_system_ready_event(eflow_proc):
    """Step 1-2: SystemReady 是 stdout 第一行"""

    assert "task_id" in first
    assert "started_at" in first


def test_02_send_dispatches_task_started_and_completed(eflow_proc):
    """Step 3: send 触发 TaskStarted + TaskCompleted"""

    send_stdin(eflow_proc, {"action": "send", "task": "test_task_1"})

        if any(e.get("event_type") == "TaskCompleted" for e in events):

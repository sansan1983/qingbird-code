## 3. 发现(Findings)

### Finding 1 ⚠️ `user_message()` 已建但 main.rs 未调用

**位置:** `crates/qbird-code/src/main.rs:124, 299`

**现象:**
```rust
eprintln!("{}", t!("status_config_load_fail", msg = e.to_string()));   // line 124
eprintln!("{}", t!("status_task_failed", msg = e.to_string()));         // line 299
```

我在 18-04 把 `EflowError::Display` 改成英文(为对齐 AGENTS.md "tracing 英文"),但这两处仍用 `e.to_string()` 拿 Display,结果:
- `zh-CN` 模式:`加载配置失败: <英文 details>` — 中英混杂
- `en-US` 模式:`加载配置失败: <英文 details>` — 完全中文

**原因:** Plan 18-04 文字只要求 "user_message() 内走 t!",没明说 call site 切换;我没改 call site。

**影响:** 实际使用 yaml + 真实 LLM 时几乎遇不到(配置不 load fail、task 也很少 fail),但违背 i18n 审计目标。

**建议修复:** 这两行换成 `e.user_message()`。**v0.2.18 不必修(plan 未要求)**,但应在 19-04 之前补一个 fix-up commit。

### Finding 2 ⚠️ 启动顺序:config load 早于 locale init

**位置:** `crates/qbird-code/src/main.rs:121-132`

**现象:** `load_config` 在 `locale::init` 之前。如果用户 `--lang en-US` 但 qingbird.yaml 损坏,错误信息走默认 locale(zh-CN)而不是 en-US。

**原因:** 需要先有 cfg 才能读 `cfg.core.language`,locale 解析需要 yaml 信息。

**影响:** 极小,plan 18-03 验收测试不触发此分支。

**建议:** 不修 — 这种"配置都读不出来"是灾难场景,i18n 在这种场景下优先级低。如果要修,需要把 `cli.lang` 单独硬编码,直接调 `locale::init(cli.lang)`,再调 `load_config`(让它用 init 过的 locale 打错误)。需要 minor 重构。

### Finding 3 ⚠️ 23 个孤儿 i18n key

**位置:** `locales/zh-CN.yml` + `locales/en-US.yml`

**现象:** 115 个 key 中 23 个定义但代码未引用:
```

err_config_parse             (历史)
err_config_read              (历史)
err_http / err_json_parse    (历史)
err_llm_init                 (历史 LlmRouter)

```

**原因:** v0.2.x 历史(被删模块的 key 残留)+ 我 18-02 留的 placeholder。

**影响:** 无功能影响,只是 yml 体积。Plan 18-06 的精神是"清理历史",但没明确要求清 i18n key。

**建议:** 下个 PR 清理(注意 en-US 同步删)。可在 18-12 后续的 fix-up commit 一起做。

### Finding 4 ⚠️ `default_model` 硬编码空串 fallback

**位置:** `crates/qbird-code/src/main.rs:140-146`

**现象:** `cfg.llm.active` 是不在白名单的 provider 时,`default_model = ""`。

**影响:** `--model <空>` 会传给 provider,导致请求失败。19-06 `EflowConfig::validate()` 会先拦截这个 case,所以 Phase 2 上线后无影响。

**建议:** 19-06 之前不动;19-06 上线后,这条 `_ => ""` 永远走不到。

### Finding 5 ℹ️ 残留中文代码注释(非 Phase 1 引入)

**位置:** `main.rs`, `react_loop/*.rs`, `doom_loop.rs`, `nudge.rs`, `subagent.rs` 等。

**现象:** AGENTS.md 要求"代码注释保持英文",但 v0.2.17 基线已有大量中文 `///` doc comment 和 `//` 行注释。

**Phase 1 引入的中文注释:** 0(我加的注释全英文:locale.rs 改写、event.rs 顶部、error.rs 文档)

**建议:** 单独 task 清理(超出 v0.2.18 范围)。

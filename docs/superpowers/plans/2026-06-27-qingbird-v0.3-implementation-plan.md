# qingbird v0.3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 qingbird 从 v0.2.17 的"声明 ship 实际未工作"状态,推进到 v0.3.0 的"日常编码助手"可用态;覆盖 22 个 work item,分 3 个版本发布。

**Architecture:** 5-crate workspace;v0.2.18 清理 + 文档(无行为变更),v0.2.19 基础设施接线(动核心循环),v0.3.0 日常可用打磨(新 user-facing 功能)。Streaming 走 `StreamFormat` 格式分发,Provider 路由走 `RuntimeOverrides` 隔离瞬时态,Profile 系统走用户级 `data_dir()/qingbird/profiles/*.yaml`。

**Tech Stack:** Rust 2024, tokio, serde, rusqlite (bundled), reqwest, `rust-i18n`, `similar = "2"` (新增)

**设计文档:** `docs/superpowers/specs/2026-06-27-qingbird-v0.3-cleanup-and-daily-usable-design.md`

## Global Constraints

从设计文档与项目 AGENTS.md 抄录:

- 所有版本号延续 0.2.x / 0.3.x 节奏,本计划不跳 1.0
- 所有用户面向字符串必须配齐 `locales/zh-CN.yml` + `locales/en-US.yml`
- 每个 PR 前必须通过 4 门禁:`cargo fmt --check` → `cargo clippy --all-targets --workspace -- -D warnings` → `cargo test --workspace` → `cargo build`
- 依赖方向严格:`qbird-code-models ← qbird-code-infra ← {qbird-code-tools, qbird-code-agents} ← qbird-code`,下层禁止引用上层
- `tracing` 日志保持英文,用户面向消息走 `t!()`
- 代码注释保持英文
- 涉及 LLM 的测试必须 dummy key + 5s timeout
- Conventional Commits,scope = 模块名(`react-loop` / `memory` / `tools` / `config` / `llm` / `cli` / `i18n` / `agents` / `docs` / `chore`)
- 共享 workspace 版本号,所有 crate 同步 bump

## Task 18-02: `/sdd confirm` 接通 `hard_gate_blocked`

**Files:**
- Modify: `crates/qbird-code/src/main.rs:485-525`
- Modify: `crates/qbird-code-agents/src/skill/sdd/proposal.rs:34-58`
- Modify: `locales/zh-CN.yml`
- Modify: `locales/en-US.yml`

**Scope:** 当前 `/sdd confirm` 打印 placeholder;改为读 session state,如果有 `pending_proposal` (来自 `/sdd run` 的 proposal output),则清除 `hard_gate_blocked` 状态并打印"已通过"消息。

**Interfaces:**
- Consumes: `SddProposal.hard_gate_blocked: bool`, session 内存中的 `pending_proposal: Option<SddProposal>`
- Produces: `/sdd confirm` 真实状态机

**State Machine:**
```
/sdd run <input>  →  pending_proposal = Some(...), hard_gate_blocked = true
/sdd confirm       →  if pending_proposal.is_some() { hard_gate_blocked = false; clear pending }
/sdd status        →  show pending + blocked status
```

**Acceptance:**
- [ ] `/sdd run X` 后 `/sdd status` 显示 `pending + blocked`
- [ ] `/sdd confirm` 后 `/sdd status` 显示 `confirmed`
- [ ] `/sdd confirm` 无 pending 时打 `interactive_sdd_no_pending` 错误(不 panic)
- [ ] 5 单测覆盖 4 状态转换 + 错误路径

**Risks:** 中(改 skill 输出消费路径)

**Est:** 1d

// react_loop/mod.rs

fn on_iteration_done(messages: &mut Vec<Message>, cm: Option<&mut ContextManager>) {
    if let Some(cm) = cm {
        if let Some(last) = messages.last() {
            cm.add_message(last);
        }
        if let Some(reminder) = cm.checkpoint_if_needed() {

    }
}
